# 项目概览

## 项目愿景

Novel Agent System 的愿景是打造一个完全自主、高度智能的小说创作与发布生态系统，让每一个有故事想要讲述的人都能借助 AI 的力量实现创作梦想。我们相信，AI 不应该取代人类创作者，而是应该成为他们最得力的助手，将繁琐的重复性工作自动化，让创作者能够专注于真正需要创造力和想象力的部分。

### 核心目标

1. **降低创作门槛** - 让没有专业写作背景的人也能创作出高质量的小说作品
2. **提升创作效率** - 将原本需要数月甚至数年的创作周期缩短到数周或数天
3. **保证作品质量** - 通过多代理协作和质量控制机制，确保作品的连贯性和专业性
4. **扩大传播范围** - 一键多平台发布，让作品触达更广泛的读者群体
5. **数据驱动优化** - 通过市场研究和读者反馈，持续优化作品内容和创作策略

### 长远愿景

我们希望在未来能够建立一个完整的 AI 小说创作社区，在这里：
- 创作者可以分享和交流创作经验
- AI 模型可以不断学习和进化
- 作品可以通过智能推荐系统找到最合适的读者
- 创作者可以获得公平的收益和认可

## 核心特性详解

### 1. 多代理协作系统

Novel Agent System 的核心是一个由 8 个专业 AI 代理组成的协作网络，每个代理都有自己独特的专业领域和职责，它们通过标准化的接口进行通信和协作，共同完成小说创作的全流程。

#### 1.1 剧情规划代理

**职责**：负责整体故事架构的设计和章节大纲的制定

**核心功能**：
- 分析和设计完整的故事弧线（开端、发展、高潮、结局）
- 制定详细的章节大纲，包括每章的主要情节、冲突点和目标
- 规划伏笔和悬念的设置与回收
- 确保整体叙事节奏的合理性
- 根据不同类型小说的特点调整剧情结构

**工作流程**：
1. 接收初始创作需求（题材、主题、篇幅等）
2. 生成多个可选的故事架构方案
3. 根据反馈进行迭代优化
4. 输出详细的章节大纲和剧情规划文档

#### 1.2 角色创建代理

**职责**：构建丰富立体的人物角色档案

**核心功能**：
- 设计主角、配角、反派等各类角色的完整设定
- 包括外貌、性格、背景、动机、成长弧线等维度
- 建立角色之间的关系网络和互动模式
- 确保角色行为的一致性和合理性
- 为每个角色创建独特的语言风格和行为习惯

**角色档案包含**：
- 基本信息（姓名、年龄、职业等）
- 外貌特征（身高、体型、发型、眼睛等）
- 性格特点（优点、缺点、怪癖等）
- 背景故事（家庭、教育、经历等）
- 动机与目标（短期目标、长期梦想、内心冲突等）
- 关系网络（与其他角色的关系）
- 成长弧线（角色在故事中的变化和发展）

#### 1.3 世界观构建代理

**职责**：创建完整自洽的世界设定和规则体系

**核心功能**：
- 设计世界的地理环境、历史背景、社会结构
- 建立魔法系统、科技体系或超自然规则
- 制定文化习俗、宗教信仰、政治制度
- 确保世界观的一致性和逻辑性
- 为故事发展提供合理的舞台和背景

**世界观设定包含**：
- 地理与环境（大陆、国家、城市、自然景观等）
- 历史与时间线（重要历史事件、时代背景等）
- 社会与政治（社会结构、政治制度、法律体系等）
- 经济与科技（经济体系、科技水平、交通方式等）
- 文化与宗教（文化习俗、宗教信仰、艺术形式等）
- 特殊规则（魔法系统、超自然能力、特殊法则等）

#### 1.4 类型专精作家

**职责**：根据不同类型小说的特点进行专业创作

系统包含 5 位类型专精作家，每位都深入研究和学习了对应类型的经典作品和写作技巧：

**科幻作家**：
- 精通硬科幻物理法则、太空探索、未来科技
- 擅长时间旅行、人工智能、星际战争等主题
- 注重科技设定的合理性和科学性
- 能够将复杂的科学概念融入精彩的故事中

**奇幻作家**：
- 精通魔法系统设计、神话元素运用、种族设定
- 擅长史诗叙事、冒险旅程、英雄成长等主题
- 善于构建宏大而充满想象力的奇幻世界
- 能够创造独特而自洽的魔法规则

**言情作家**：
- 精通情感弧线设计、对话技巧、关系动态
- 擅长情感张力营造、情节节奏把控
- 注重人物心理描写和情感细节
- 能够创造令人心动的爱情故事

**历史作家**：
- 精通时代考证、文化背景研究、历史事件还原
- 擅长再现历史风貌、社会制度、语言风格
- 注重历史细节的准确性和真实性
- 能够将虚构故事完美融入历史背景

**军事作家**：
- 精通战术策略、武器装备、指挥体系
- 擅长战场描写、军事术语运用、军队建设
- 注重军事行动的合理性和逻辑性
- 能够创造紧张刺激的战争场景

#### 1.5 编辑审核代理

**职责**：负责作品的质量控制和修改建议

**核心功能**：
- 检查作品的语法、拼写、标点等技术性错误
- 评估作品的叙事节奏、情节发展、人物塑造
- 发现逻辑漏洞、情节矛盾、人物行为不一致
- 提供具体的修改建议和优化方向
- 确保作品符合目标平台的内容规范

**审核维度**：
- 技术质量（语法、拼写、标点等）
- 叙事质量（节奏、结构、连贯性等）
- 人物质量（一致性、立体性、发展性等）
- 情节质量（逻辑性、吸引力、合理性等）
- 内容合规性（平台规范、敏感内容等）

#### 1.6 发布代理

**职责**：负责多平台一键发布和格式转换

**核心功能**：
- 支持多个主流小说发布平台（Wattpad、Royal Road、Amazon KDP）
- 自动进行格式转换和平台适配
- 管理作品元数据（标题、简介、标签、封面等）
- 定时发布和章节更新管理
- 发布状态监控和错误处理

#### 1.7 市场研究代理

**职责**：负责市场趋势分析和竞品调研

**核心功能**：
- 分析当前热门题材和流行趋势
- 研究成功作品的特点和规律
- 监控竞品动态和市场变化
- 提供选题建议和创作方向
- 分析读者偏好和阅读习惯

**研究内容**：
- 热门题材和类型趋势
- 成功作品案例分析
- 读者评论和反馈分析
- 平台算法和推荐机制研究
- 市场竞争格局分析

#### 1.8 读者互动代理

**职责**：负责读者评论分析和自动回复

**核心功能**：
- 收集和分析读者评论和反馈
- 识别读者的喜好、建议和批评
- 生成合适的回复内容
- 维护与读者的良好互动关系
- 将读者反馈反馈给创作团队

### 2. OpenViking 内存系统

为了支持长篇小说的创作，系统开发了专门的 OpenViking 内存系统，确保在长达 50-300 万字的创作过程中保持上下文的连贯性和一致性。

#### 2.1 角色记忆模块

**功能**：
- 存储所有角色的完整档案信息
- 记录角色在故事中的所有行为和对话
- 追踪角色的成长和变化
- 确保角色行为和语言风格的一致性
- 快速检索角色相关信息

#### 2.2 剧情记忆模块

**功能**：
- 记录已发生的所有情节和事件
- 追踪伏笔的设置和回收
- 维护时间线和事件顺序
- 检测剧情矛盾和逻辑漏洞
- 提供剧情回顾和摘要

#### 2.3 世界观记忆模块

**功能**：
- 存储完整的世界观设定
- 记录世界中的重要地点、组织、物品
- 确保世界观规则的一致性应用
- 快速检索世界观相关信息
- 检测世界观设定的矛盾

#### 2.4 一致性检查引擎

**功能**：
- 自动检测角色行为不一致
- 自动检测剧情逻辑矛盾
- 自动检测世界观设定冲突
- 提供修复建议和方案
- 生成一致性报告

### 3. 全自动化工作流

系统支持从选题到发布的完全自动化工作流，可以按照预设的时间表自动执行创作和发布任务。

#### 3.1 任务调度系统

基于 Celery 和 Redis 的分布式任务调度系统：
- 支持定时任务和周期性任务
- 支持任务依赖和工作流编排
- 支持任务重试和错误处理
- 支持任务监控和日志记录
- 支持分布式部署和水平扩展

#### 3.2 健康监控与告警

- 实时监控系统各组件的运行状态
- 自动检测和处理异常情况
- 支持多种告警方式（邮件、短信、Webhook）
- 提供详细的性能指标和统计数据
- 支持历史数据查询和分析

## 设计理念

### 1. 代理模式

所有 AI 代理都继承自统一的 `BaseAgent` 抽象基类，实现标准化的 `async execute()` 方法。这种设计的优势：
- 统一的接口，便于代理编排和协作
- 易于添加新的代理类型
- 便于测试和模拟
- 清晰的职责分离

### 2. 异步优先

系统全面采用异步编程模式：
- 所有 I/O 操作（LLM 调用、文件读写、网络请求）都是异步的
- 使用 `async/await` 语法，代码清晰易读
- 支持高并发和高效的资源利用
- 适合处理多个代理的并行协作

### 3. 结果模式

所有操作都返回统一的 `AgentResult` 数据结构：
```python
@dataclass
class AgentResult:
    success: bool
    data: dict[str, Any]
    errors: list[str] = field(default_factory=list)
```

这种模式的优势：
- 统一的错误处理方式
- 便于链式调用和结果传递
- 清晰的成功/失败状态
- 丰富的错误信息

### 4. 配置模式

使用 Pydantic Settings 进行配置管理，结合 `@lru_cache` 实现单例访问：
- 类型安全的配置验证
- 支持环境变量和配置文件
- 集中式配置管理
- 易于测试和mock
